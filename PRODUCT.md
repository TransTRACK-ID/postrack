# Product

## Register

product

## Users

API developers, QA engineers, and backend teams who build, test, mock, and document APIs. They use the tool daily in their workflow, often juggling multiple workspaces, environments, and collections. The primary task on any screen is managing or executing API requests.

## Product Purpose

A full-stack workspace for building, testing, mocking, documenting, and sharing APIs. Success looks like a developer reaching for this tool instead of Postman/Insomnia because it feels faster, lighter, and more pleasant to use.

## Brand Personality

Playful, approachable, energetic. The interface should feel like a well-designed toy: precise, satisfying, and never stiff. Three words: **playful**, **precise**, **energetic**.

## Anti-references

- Avoid the heavy, corporate feel of enterprise API gateways.
- Avoid the sterile gray-on-gray exhaustion of default developer tools.
- Avoid the "everything is a card" SaaS template look.

## Design Principles

1. **Precision with personality**: Every interaction should feel snappy and intentional, but never robotic.
2. **Show, don't tell**: Status, methods, and state should be communicated through color and shape before text.
3. **Hierarchy through scale and weight**: Avoid flat scales; use dramatic size differences to guide attention.
4. **Reward the eye**: Micro-interactions, subtle depth, and color accents make daily use enjoyable.
5. **Respect the workflow**: The sidebar is a navigation instrument, not a destination. It should be compact, scannable, and out of the way.

## Accessibility & Inclusion

- WCAG 2.1 AA baseline
- Respect `prefers-reduced-motion`
- Ensure sufficient contrast for all method/status color badges
